📥 How to import into TestRail — step by step ▼
⭐ Recomendado: usa el botón "Descargar XML" — el formato XML de TestRail no requiere mapeo de columnas. TestRail lo reconoce automáticamente: en Import Cases selecciona XML, sube el archivo y listo. Usa CSV solo si prefieres una hoja de cálculo o necesitas editar los casos antes de importar.
Importar con XML (recomendado)
  1. Descarga el archivo XML con el botón ⬇ Descargar XML.
  2. En TestRail, ve a Test Cases → ícono de engranaje → Import Cases.
  3. Selecciona el formato XML, sube el archivo y haz clic en Import.
  4. TestRail detecta secciones, template y campos personalizados automáticamente. ¡Listo!
Importar con CSV (alternativa)
  1. Download the CSV using the button above.
  2. In TestRail, go to Test Cases → click the gear/settings icon → Import Cases.
  3. Upload the CSV. You should see "CSV file uploaded successfully". Click Next.
  4. Select "Configure new mapping" and click Next.
  5. Row Layout: select "Test cases use a single row". For the Template field, confirm it maps to Behaviour Driven Development.
  6. Map each CSV column to the TestRail field using the table below. Any field not listed must be set to "-- Ignore --".
  7. Save the mapping with a name (e.g. "BDD Generator") — next time select "Load mapping from configuration file" to skip this step.
  8. Click Next → Import and verify the imported cases.

Fields to set as "-- Ignore --"

The TestRail dropdown shows many more fields. Set all of the following to -- Ignore -- — they are not used with the Behaviour Driven Development template:

TestRail field in dropdown Why ignore
StepsSteps fields belong to the Test Case (Steps) template. BDD template uses BDD Scenarios instead.
Steps (Additional Info)Same — belongs to step-by-step template only.
Steps (Expected Result)Same — belongs to step-by-step template only.
Steps (Step)Same — belongs to step-by-step template only.
Steps (References)Per-step reference links — not included in the CSV.
Steps (Shared Step ID)Shared/reusable steps feature — not used here.
Test DataPer-step field; test data is embedded inside the BDD scenario.
Use CaseOptional field not populated by the generator.
Labels / Modules / Automation TypeMetadata fields — not included in the CSV.
Automation Review / Domain / ImplementationPresent only in CSV from Batch XML tab. Ignore when importing single-case CSV.
Release / Project / Section Description / Section HierarchyMetadata fields — not included in the CSV.
